# AI Document Management System - Project Context

## Project Overview
This project is an AI-powered Document Management System (Milestone 1 MVP). It allows users to upload documents and images (PDF, JPG, PNG), automatically extracts text from them using Optical Character Recognition (OCR), and provides an interface to manage, search, and view the extracted results.

## Current Progress
The Milestone 1 MVP is fully operational. Both the backend API and the interactive frontend interface have been implemented and integrated successfully. A dedicated OCR pipeline is in place to handle text extraction for both single images and multi-page PDFs.

## Completed Features
- **File Uploads**: Drag-and-drop or click-to-upload interface with file size and type validation (max 20 MB; PDF, JPG, PNG).
- **OCR Processing**: Automated text extraction using PaddleOCR and pdf2image.
- **State & Data Management**: Real-time progress tracking of uploaded files and persistent storage using SQLite.
- **Document Dashboard**: A grid-based UI showing document status, file size, page count, and text previews.
- **Document Actions**: 
  - View full extracted text in a clean modal view.
  - Delete documents and their associated local files.
  - Copy extracted text to the clipboard.
  - Export text as `.txt`.
  - Download the original uploaded file.
- **Search Functionality**: Client-side filtering of documents by filename or extracted text content.

## Tech Stack Used
- **Backend API**: Python 3.x, FastAPI, Uvicorn (ASGI server)
- **Database**: SQLite, SQLAlchemy ORM
- **OCR Engine**: PaddleOCR (configured for English by default), `pdf2image` for PDF processing.
- **External Dependencies**: Poppler (Windows binaries extracted locally in `E:\fiverocr\poppler` and mapped in `ocr_service.py` to enable PDF processing).
- **Frontend**: Vanilla HTML5, CSS3 (Custom styling, modern layout), Vanilla JavaScript (DOM manipulation, Fetch API).

## Current Issues
- None currently. (Earlier issues with Poppler path resolution on Windows during PDF processing were successfully resolved by downloading the binaries locally and updating `ocr_service.py`).

## Next Steps / Future Enhancements
- Support for selecting different languages for OCR (e.g., Arabic, Chinese).
- Improve error handling and robust OCR retry mechanisms.
- Add user authentication and personalized document spaces.
- Migrate local file storage to a cloud provider (e.g., AWS S3) for scalability.
- Implement advanced document search (e.g., Elasticsearch or vector search).
